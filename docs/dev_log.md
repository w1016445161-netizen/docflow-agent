## v0.4-step1: 统一异常处理与结构化日志

本次更新为后端增加了统一异常处理、request_id 请求追踪和结构化 JSON 日志。

主要改动：
1. 新增 ApiResponse 响应模型，用于统一异常响应。
2. 新增 AppException 业务异常基类，并按文件解析、OCR、LLM 调用划分错误码。
3. 在 FastAPI 中间件中为每个请求生成 request_id，并通过 contextvars 在调用链中透传。
4. 在文档上传、PDF 解析、OCR、LLM 调用、文档问答等关键路径记录结构化日志。
5. 日志同时输出到控制台和文件，便于本地调试和后续接入日志收集系统。

当前边界：
1. 正常成功响应暂保持原有结构，未统一改为 ApiResponse。
2. 日志系统目前用于本地排查，尚未接入 ELK / Loki。
3. 后续计划增加文档状态管理和数据库持久化。
